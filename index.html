Introducing Sandbox

從 production repo 萃取出的獨立可執行開發環境

Production Repo

需要 VPN、Auth token、Feature flags、Backend services、完整開發環境

Sandbox Repo

零門檻啟動：yarn dev 就能跑。Auth/flags/permissions 全部 mock 掉

sandbox-entry.jsx 自動 shim 所有外部依賴，讓 app 以為自己在正常的 V1 環境裡

Architecture

Subtraction, not Translation

Production Repo (app-spm-ui)
├── src/           ──→  直接複製
├── proxy/         ──→  直接複製 (mock server)
├── vite.config    ──→  直接複製
├── Auth system    ──→  ✂️ 移除，用 sandbox-entry.jsx 替代
├── Feature flags  ──→  ✂️ 全部開啟
├── Permissions    ──→  ✂️ 全部授權
└── Backend deps   ──→  ✂️ 用 mock server 取代

= Sandbox Repo (sandbox-app-spm-ui)
  yarn dev → 立刻可跑

How It Works

sandbox-entry.jsx 取代 src/index.js，移除所有外部依賴

// 1. 模擬 Vision One parent frame
window.parent._CURRENT_ENV = 'STG';
window.parent._CURRENT_REGION = 'US';

// 2. Mock auth tokens
localStorage.setItem('uic_userGUID', 'sandbox-user');

// 3. 開啟所有 feature flags
cookie.set('flag_almalinux10', '1');

// 4. 啟動真正的 app
import { renderApp } from './src/app';
renderApp();

Designer 不需要理解這些 — 只需要 yarn dev

One more thing...

The Missing Piece

UI Flow Docs 描述了畫面長什麼樣

但缺少了行為怎麼轉換

Flow Docs 擅長

Layout、欄位、元件名稱、
操作步驟（線性流程）

Flow Docs 缺少

狀態之間怎麼轉換、
哪些操作在哪個狀態下被禁止、
分支邏輯的完整圖

XState Statecharts

用狀態機精確描述 UI 行為

clean: {
  on: {
    EDIT_FIELD: 'dirty',
    CLICK_CANCEL: 'policyList'
  }
},
dirty: {
  on: {
    CLICK_SAVE: 'validating',
    CLICK_CANCEL: 'discardConfirm'
  }
},
discardConfirm: {
  on: {
    CONFIRM: 'policyList',
    CANCEL: 'dirty'
  }
}

Hierarchical States

狀態可以巢狀 — 每個 capability tab 各自有 idle/editing/saving

Parallel States

多個獨立狀態同時運行 — form 狀態和 priority tab 狀態互不影響

Guards

條件守衛 — 只有 isDirty && hasPermission 才能 Save

Demo: SPM Edit Policy

三個並行狀態機在同一個畫面上

1. Form State

clean → dirty → validating → saving
+ NCIE confirm 分支
+ Discard confirm 分支

2. Priority Tabs

idle ↔ renaming
renaming 時其他 tab disabled
不能同時 add + rename

3. Selection Modals

closed ↔ groupSelection
closed ↔ osSelection
互斥：一次只開一個
